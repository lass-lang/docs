@(prop) Property Accessor - Extra Cases

These test cases cover edge cases and implementation details for @(prop) that complement the main axiom file.

valid: doesn't cross @layer boundaries - preserved

@(prop) stops at at-rule boundaries. Properties declared in one @layer are not visible to another.

@layer base {
  .box {
    color: blue;
  }
}

@layer utilities {
  .box {
    background: @(color);
  }
}
@layer base {
  .box {
    color: blue;
  }
}

@layer utilities {
  .box {
    background: @(color);
  }
}

valid: parent scope lookup inside {{ }}

@(prop) inside {{ }} can resolve from parent scope.

.parent {
  border: solid;
  .child {
    outline: {{ @(border) }};
  }
}
.parent {
  border: solid;
  .child {
    outline: solid;
  }
}

valid: multiple @(prop) in one {{ }} expression

Multiple @(prop) references can appear in a single {{ }} expression.

.box {
  width: 100;
  height: 50;
  content: {{ @(width) + " x " + @(height) }};
}
.box {
  width: 100;
  height: 50;
  content: 100 x 50;
}

valid: math with multiple @(prop) references

Arithmetic operations can use multiple @(prop) values.

.box {
  padding: 10px;
  margin: 5px;
  total: {{ parseInt(@(padding)) + parseInt(@(margin)) }}px;
}
.box {
  padding: 10px;
  margin: 5px;
  total: 15px;
}

valid: string operations with @(prop)

JavaScript string methods work on @(prop) values.

.box {
  color: blue;
  content: {{ @(color).toUpperCase() }};
}
.box {
  color: blue;
  content: BLUE;
}

valid: template literal with @(prop)

@(prop) works inside JavaScript template literals within {{ }}.

.box {
  color: blue;
  {{ `border-color: ${@(color)};` }}
}
.box {
  color: blue;
  border-color: blue;
}

valid: multiple @(prop) concatenated

Multiple @(prop) values can be concatenated with string operators.

.box {
  width: 100px;
  height: 50px;
  content: {{ "Size: " + @(width) + " x " + @(height) }};
}
.box {
  width: 100px;
  height: 50px;
  content: Size: 100px x 50px;
}

valid: @(prop) in CSS context and JS context

Same property accessed in CSS context (raw substitution) and JS context (as quoted string for expression evaluation).

.box {
  color: blue;
  border-color: @(color);
  background: {{ @(color) }};
}
.box {
  color: blue;
  border-color: blue;
  background: blue;
}

valid: nested scope @(prop) inside {{ }}

@(prop) inside {{ }} resolves from parent scope through nesting.

.parent {
  padding: 10px;
  .child {
    margin: {{ parseInt(@(padding)) * 2 }}px;
  }
}
.parent {
  padding: 10px;
  .child {
    margin: 20px;
  }
}

valid: quoted value escaping

Values containing quotes are properly escaped when used in {{ }}.

.box {
  font-family: "Arial";
  content: {{ @(font-family) }};
}
.box {
  font-family: "Arial";
  content: "Arial";
}

valid: newline in value escaping

Values containing newlines are properly escaped when used in {{ }}.

.box {
  content: "line1
line2";
  other: {{ @(content) }};
}
.box {
  content: "line1
line2";
  other: "line1
line2";
}

valid: backslash and quote escaping

Values with backslashes and quotes are preserved correctly.

.box {
  content: "path\\to\"file\"";
  other: {{ @(content) }};
}
.box {
  content: "path\\to\"file\"";
  other: "path\\to\"file\"";
}

valid: @(prop) resolves before {{ }}

@(prop) in CSS context resolves at compile time (Phase 1), before {{ }} evaluates at runtime (Phase 2).

---
const color = "green";
---
.box {
  border: solid;
  outline: @(border);
  background: {{ color }};
}
.box {
  border: solid;
  outline: solid;
  background: green;
}

valid: @(prop) after {{ }} in same block

@(prop) appearing after a {{ }} expression in the same block still resolves correctly.

.box {
  color: blue;
  margin: {{ 10 }}px;
  border-color: @(color);
}
.box {
  color: blue;
  margin: 10px;
  border-color: blue;
}

valid: @(prop) at root level after {{ }}

@(prop) works at root level (no selector block) after {{ }}.

color: blue;
{{ 'injected' }}
border: @(color);
color: blue;
injected
border: blue;

invalid: @(prop) not found inside {{ }} causes runtime error

When @(prop) is not found and is inside {{ }}, it's preserved as @(prop) which is invalid JavaScript, causing a runtime error.

.box {
  color: {{ @(nonexistent) }};
}
Invalid or unexpected token
