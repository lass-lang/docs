// Single-Line Comments

Lass adds // single-line comments to CSS. Everything from // to the end of the line is stripped from the output.

Standard CSS /* */ comments still work and are preserved in the output — nothing changes there.

The scanner only detects // outside of CSS string literals, url() values, and /* */ comments. A // inside "..." or url(...) is just text.

valid: basic comment stripped

p {
  // this comment is stripped
  color: red;
}
p {
  
  color: red;
}

valid: inline comment stripped

// strips everything from the // to end of line. The declaration before it is preserved.

p {
  color: red; // this is stripped
}
p {
  color: red; 
}

valid: CSS comments preserved

/* */ comments are standard CSS — they pass through into the output.

/* preserved */
p {
  color: red; /* also preserved */
}
/* preserved */
p {
  color: red; /* also preserved */
}

valid: both comment styles together

// stripped, /* */ preserved — they coexist.

// this line disappears
/* this stays */
p {
  color: red; // gone
  background: blue; /* stays */
}

/* this stays */
p {
  color: red; 
  background: blue; /* stays */
}

valid: // inside a CSS string is not a comment

The scanner skips // detection inside string literals.

a {
  content: "https://example.com";
}
a {
  content: "https://example.com";
}

valid: // inside url() is not a comment

The scanner skips // detection inside url().

.bg {
  background: url(https://example.com/image.png);
}
.bg {
  background: url(https://example.com/image.png);
}

valid: // inside a /* */ comment is not a comment

// inside a /* */ block is just part of the CSS comment text. The /* */ takes precedence.

/* This contains // but it's all one CSS comment */
p {
  color: red;
}
/* This contains // but it's all one CSS comment */
p {
  color: red;
}

valid: comment-only file produces empty output

// nothing but a comment

valid: // comments in no-separator pure CSS file

// stripping works regardless of whether the file has a --- separator. A pure CSS file (no preamble) with // comments gets them stripped just the same.

// top-level comment
p {
  color: red; // inline comment
}

p {
  color: red; 
}

invalid: unclosed /* */ breaks comment-state detection

An unclosed /* */ comment means the scanner enters comment-state and never exits. Everything from /* onward is consumed as a comment, and the output is truncated. The transpiler detects this at end-of-file and reports an error.

p {
  color: red;
}
/* this comment never closes
.box {
  padding: 8px;
}
Unclosed /* comment
